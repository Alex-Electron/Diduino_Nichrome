# Diduino Nichrome

[English](./README.md) · **Русский**

Прошивка и браузерный программатор для советского биполярного ППЗУ **КР556РТ4** (256×4 бита, однократно программируемое).

Это продолжение открытого проекта **Diduino**. Железо то же самое; новое здесь — переписанное приложение, которое работает прямо в браузере, и алгоритм прожига, сделанный так, чтобы не убить чип, когда что-то идёт чуть-чуть не так.

## Зачем я переписал софт

Штатный Diduino работает. Три вещи всё-таки заставили переделать хост и прошивку:

1. **Веб-страница, которую просто открываешь.** Не надо качать, ставить и обновлять Qt-приложение. Один самодостаточный HTML-файл, открывается прямо с диска в Chrome или Edge и общается с платой через Web Serial. Больше ничего настраивать не нужно.

2. **Более аккуратная и надёжная прошивка.** Оригинальный скетч — это один движок чтения/записи. Я хотел, чтобы сам прожиг был осторожным: проверял чип до того, как его трогать, отказывался от битой передачи, выставлял напряжение программирования сам, а не надеялся, что я не забуду, и всегда оставлял плату в безопасном состоянии. КР556РТ4 однократное — одна ошибка губит чип, поэтому прошивка должна ловить очевидные промахи, а не послушно их выполнять.

3. **Soak-импульсы.** Перемычка, которая едва открылась, может прочитаться как прожжённая, а потом «прорасти» обратно. После того как бит взялся, прошивка добивает в него несколько импульсов (даташит разрешает 40–100), чтобы перемычка пережглась полностью, а не на грани. Это вышло из копания в спецификации и есть главная новая идея на стороне прожига.

## Чип: КР556РТ4

Советское биполярное ППЗУ с нихромовыми перемычками, 256 слов по 4 бита, аналог 82S129.

- Перемычки **односторонние**: бит может перейти 0 → 1, но никогда обратно. Чистый чип читается как одни нули.
- Для программирования нужно повышенное напряжение (~12.5 В) на ноге данных, при Vcc = 5 В.
- Чип одноразовый — на него ровно одна попытка. Смысл этого проекта в том, чтобы эта попытка удалась.

## Быстрый старт

1. **Открыть приложение.** Открой [прошивальщик по ссылке](https://alex-electron.github.io/Diduino_Nichrome/diduino_nichrome.html) в Chrome или Edge (либо `docs/diduino_nichrome.html` локально). Нажми CONNECT и выбери COM-порт платы.
2. **Прошивка.** Если на плате не Diduino_Nichrome (или версия старая), операции блокируются, и приложение предлагает **[ ⟳ FLASH NANO ]** — прошивка заливается прямо из браузера через Web Serial (без Arduino IDE): читается сигнатура чипа и подбирается нужная сборка (ATmega328P или ATmega168). Можно и по-старому — залить `firmware/Diduino_Nichrome/Diduino_Nichrome.ino` из Arduino IDE самому. После этого в шапке появится версия прошивки, а текущее Vpp считается автоматически.
3. **Читать или жечь.**
   - `READ` считывает все 256 нибблов и показывает hex и CRC-32.
   - `LOAD` открывает `.bin`; тот же hex-вид и CRC для файла.
   - `BURN` программирует чип. Запускается только если чип совместим и контрольная сумма передачи совпала, после чего сверяет каждую ячейку на плате.

Интерфейс по умолчанию русский; кнопка справа вверху переключает на английский.

## Прожиг по шагам («Nichrome»)

Прожиг устроен так, что обычные способы загубить чип ловятся до того, как напряжение дойдёт до перемычки:

| Этап | Что делает |
|------|------------|
| **Передача + CRC-32** | Хост шлёт 256 байт; плата считает CRC-32, и хост обязан прислать свою ожидаемую CRC. Не совпало — прожиг не запускается. Битый или не тот файл не пройдёт. |
| **Проверка совместимости** | Сначала чип читается. Если где-то уже стоит 1 там, где образу нужен 0 (на фьюз-ППЗУ это не отменить), прожиг прерывается со списком конфликтов. Чистый чип — или дожиг того же образа на частично прошитый — проходит. |
| **Проверка напряжения** | Плата сама выставляет уровень программирования (p1, ~12.5 В) и измеряет его. Жечь перемычку на неверном напряжении только потому, что уровень забыли, она не станет. |
| **Побитовая сверка** | Каждый бит записывается (прожигается) и читается; импульсы прекращаются, как только бит взялся. |
| **Soak** | Несколько добивочных импульсов после взятия бита, чтобы перемычка пережглась до конца. |
| **Сверка в покое** | Напряжение буста снижается до минимума, весь чип перечитывается и сверяется с образом. |
| **Безопасное состояние** | На любом выходе — успех, отмена или ошибка — плата заканчивает без импульса программирования, буст на минимуме, буфер порта вычищен. |

## Serial-протокол (115200 8N1, команды завершаются Enter)

| Команда | Значение |
|---------|----------|
| `?` | помощь |
| `v` | прочитать напряжение программирования (Vpp) |
| `p<n>` | уровень буста 0–7 (p1 ≈ 12.5 В для РТ4) |
| `I<n>` | макс. импульсов на бит (по умолчанию 1000) |
| `L<n>` | длительность импульса, мкс (по умолчанию 40) |
| `D<n>` | заполнение %, задаёт паузу между импульсами (по умолчанию 10) |
| `S<n>` | soak-импульсы после взятия бита (по умолчанию 50, потолок 128) |
| `R` | прочитать весь чип, 256 байт |
| `B` | прожиг: 256 байт, затем CRC-гейт, проверки и сверка |
| `s` | безопасное состояние |

Есть и диагностический набор для наладки платы (`a c m d e k W`) — проверка адресной цепочки, мукса, линии CS и одиночного импульса.

## Калибровка напряжения

Плата читает Vpp через делитель на A4. Константы в прошивке (`rTop`, `rBottom`, `AREF_V`) подобраны так, чтобы показание `v` совпадало с мультиметром на этой конкретной плате. Для другой платы выстави уровень p7, измерь реальное напряжение и подгони `AREF_V`, пока `v` не сойдётся.

## О безопасности

КР556РТ4 однократное. Прочти один раз:

- Прожиг **необратим**. Сначала перепроверь файл и чип.
- Высокое напряжение доходит до чипа только во время самих импульсов программирования (через линию WRITE). Выставление уровня или чтение чипа не подаёт его на панельку.
- Для единственного хорошего чипа сначала прогони на жертвенном чистом, считай обратно и сверь CRC с исходником, прежде чем жечь основной.

## Планы

- **Защита от перегрева (DS18B20).** В оригинальном Diduino были пины под датчик температуры DS18B20. С установленным датчиком прошивка могла бы следить за температурой чипа или силового транзистора во время прожига и останавливаться при перегреве. Особенно это к месту с soak-импульсами, которые греют перемычку и ключ. На текущей плате датчик не распаян, поэтому оставлено на будущую ревизию.
